# 分阶段输入格式（推荐给用户）

目标：用户每次输入都“轻量”，但能被系统稳定消费；并且从第 2 章开始强调“下一章目标 + 用户问题”。

> 规则：一次对话只产出一章（见 `constraints.md`）。

---

## 阶段 0：启动（首次创建书系）

建议用户输入（3 行足够）：

```
开始一本新书：
学科/主题：生物细胞
调研方向：细胞结构与细胞器分工（动物细胞为主），面向零基础
每章学习时长：10分钟
```

系统内部默认（用户不填也要有）：
- 每章学习时长：10 分钟（最低 5）
- 输出密度：≥ 当前示例第 1 章的 3 倍信息量
- 每章交互：≥ 2 种（Tabs + Collapse 至少）
- 推荐固定模块：步骤推演（Stepper）/ 调研摘录与来源（折叠）/ 迁移到“看得见的现象”
- 分享能力：除“复制链接”，还应给出“可嵌入方式”（例如 iframe/JS 嵌入代码）

---

## 阶段 1：继续写下一章（从第 2 章开始）

建议用户输入：

```
下一章目标：细胞膜如何控制物质进出
用户问题：为什么脂溶性分子更容易穿膜？水为什么不行？
```

如果用户不输入“下一章目标”：
- 系统自动使用上一章“下一章预告”的内容作为目标，并复用既定调研方向补调研。

如果用户不输入“用户问题”：
- 系统跳过问答模块，但仍要输出“自检/练习”。

无论用户是否给额外输入，本章仍建议包含：
- 迁移到“看得见的现象”（≥3条）
- 调研摘录与来源（折叠，≥2个来源）
- 至少 2 种有意义交互（例如 Tabs + Stepper + Selector/下拉选择器 + Checklist）

---

## 阶段 2：共读与修订（不产新章，产补丁）

建议用户输入：

```
共读修订：
我卡住的点：线粒体到底做什么
希望补充形式：误区→纠正→例子→自测
```

系统输出：
- 更新当前章节 HTML（补丁式）
- 在章节中新增“修订记录/变更说明”（折叠）
