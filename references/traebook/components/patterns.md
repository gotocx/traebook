# 交互模式（TraeBook 专用）

目标：充分利用 HTML 的交互性，但每一个交互都必须提升理解，不做无意义组件堆砌。

## 1) 分层阅读（必须具备）

### 模式：Tabs（直觉 / 规则 / 工程）
用途：同一概念分层解释，支持“1 分钟粗读”和“10 分钟深读”。  
规则：
- 默认打开“直觉”
- “工程”允许折叠进阶内容

### 模式：Collapse（误区 / 答案 / 扩展）
用途：降低干扰，保持版面整齐。  
规则：
- 默认折叠误区、答案
- 只在用户需要时展开

## 2) 步骤推演（适合源码/流程/算法/操作指南）

### 模式：Stepper（上一页/下一步/播放）
用途：让读者知道“现在在哪里、下一步是什么、为什么要这样”。  
必须包含：
- 当前步骤标题
- 本步骤解释（1-2 句）
- 关键变量/关键对象（可选）

## 3) 练习闭环（每章至少 2 题）

### 模式：Quiz（题目 → 提示 → 答案）
规则：
- 提示不直接给答案
- 答案要解释“为什么”，不是只给结论
- 题卡标题必须是问题本身，读者不展开也能看到题目
- 展开区域才放提示、答案和解析；禁止把问题藏在展开区，把“练习1/查看答案/答案解析”放在卡片标题
- 如果使用 `<details>`，`<summary>` 写完整问题，展开内容按“提示 → 答案 → 为什么”组织

推荐结构：

```html
<details class="tb-question-card">
  <summary>为什么脂溶性分子更容易穿过细胞膜？</summary>
  <div class="tb-answer">
    <p><strong>提示：</strong>先想细胞膜的主要材料。</p>
    <p><strong>答案：</strong>因为膜的磷脂双分子层内部偏疏水，脂溶性分子更容易进入这一区域。</p>
    <p><strong>为什么：</strong>这不是“分子小就能过”，而是溶解性、电荷和通道机制共同决定。</p>
  </div>
</details>
```

### 模式：Checklist（可复述测试）
规则：
- 3 问，用户可以勾选“我能复述/我还不会”
- 如果勾选“不会”，引导回到对应段落（锚点）

## 4) 信息组织（让章节更整齐）

### 模式：Info Cards（关键文件/关键术语/要点）
用途：把“必须记住的”独立成卡片，提高扫读效率。

### 模式：Callout（注意/陷阱/结论）
用途：把关键结论突出，减少读者漏掉。

## 5) 分享与复现（章节级）

### 模式：Share Bar
至少支持：
- 复制当前章节链接（含 chapterId）
- 可选：复制 state（如 step=3 / tab=rules）

> Share 的目标是“别人打开能看到同一状态”，不是社交按钮。
