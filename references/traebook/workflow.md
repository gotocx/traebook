# TraeBook 生成工作流（强制）

本文件把 TraeBook 的“用户很简单、机制很复杂”落成**可执行流程**，用于约束每次对话如何产出一章，并在**用户没有额外输入**时如何自动推进下一章。

> 关联硬规则：`constraints.md`（一次对话只产出一章 / 默认 10 分钟密度 / 无输入时下一章信息量 ≥ 上一章 3 倍）

---

## 0. 输入形态（允许极简）

用户可能只说：
- “开始 / 继续”
- “我想学习 X”

系统必须从已有书系状态（manifest/上一章预告/既定调研方向）中补全默认值，然后进入下面流程。

---

## 1. 每次对话的输出范围（一次对话只产一章）

每次对话**最多**输出：
1) 1 个章节 HTML（`book/chapters/<id>.html`）
2) 必要时更新：`book/manifest.json`（加入新章节路由与元信息）
3) 必要时更新：`book/index.html`（目录入口可点击到新章节）

禁止一次生成多章正文（见 `constraints.md`）。

### 1.1 章节链接回写（强制）

每次新增或追加章节时，必须同步维护全书导航：
- `index.html` 必须展示 manifest 中所有已生成章节的可点击链接，至少覆盖 `ch1` 与已生成的 `ch2`
- 如果下一章已经规划但尚未生成，home 可以显示禁用态/预告态，但不能完全缺席
- 新增 `chapters/ch2.html` 后，必须回写 `chapters/ch1.html`，在顶部导航或“下一章桥接”后加入指向 `ch2.html` 的“下一章”链接
- 新增 `chapters/chN.html` 后，必须回写 `chapters/ch(N-1).html` 指向新章节，并在新章节中提供返回上一章与返回目录链接
- 发布前从 home 点击进入每一章，再从每章点击上一章/下一章/返回目录，确认所有相对路径可用

---

## 2. 章节生成的“内部步骤”（必须执行）

### 2.1 复用“既定调研方向”（有则继承）
从以下位置寻找“调研方向/下一章预告/用户未解决问题”：
- 最近一章章节页的“下一章预告”
- 用户最近提出的问题（若存在）
- 书籍 manifest 的 tags/章节主题

形成本章的 **3 条学习目标**（可验收）。

### 2.2 强制调研（去幻觉）
至少找到 **2 个可公开访问**的高质量来源（例如：教材/大学课程/权威科普平台/NCBI Bookshelf 等），并提取：
- 关键结论（1–3 条）
- 必要定义/边界条件
- 典型误区（至少 1 条）

### 2.3 写进章节（“调研摘录与来源”折叠区）
必须在章节页中新增一个折叠区块：
- 标题固定：`调研摘录与来源（折叠）`
- 每个摘录必须绑定外链（`<a href="...">来源</a>`）
- 摘录以“结论 → 解释 → 适用范围/边界”格式呈现

### 2.4 交互编排（至少 2 种，有意义）
交互必须回答至少一个问题：Step / Why / What-if / Check / Pitfall（见 `constraints.md`）。

建议组合（根据主题选）：
- Tabs（直觉/规则/深入）
- Stepper（把流程变成可推演步骤）
- Selector（改变参数，立即得到解释反馈）
- Checklist + 小测（折叠答案）
- ShareBar（可复现分享状态）

另外强烈建议加入一个“落地迁移”模块：
- 标题建议：`把本章迁移到“看得见的现象”`
- 要求：≥3 条（现象 → 用本章模型解释）

### 2.4.1 无插图动态降级（必须不留空）
如果本章插图服务不可用、没有公开 URL、用户禁用插图，或运行环境无法托管图片：
- 不输出空 `<figure>`、broken `<img>`、base64 或虚构 CDN URL
- 用“概念地图 / Stepper / 误区 vs 正解 / 参数小实验 / 关键术语矩阵”之一替代插图
- 替代模块必须继续服务原插图 purpose，并能回答 Step / Why / What-if / Check / Pitfall 中至少一个问题
- 在 HTML 注释里记录 fallback reason，方便后续恢复插图

### 2.5 完成度与激励（本地即可）
建议每章都有：
- 完成度（来自 Checklist 选中比例）
- XP/积分（localStorage，仅用于本地学习激励）

---

## 3. “用户没输入也要继续”模式（Auto-Research-Next）

当用户只说“继续/开始”且没有给出额外目标时：

1) **默认选题**：使用上一章预告作为“下一章主题”；若没有预告，则沿书系主线推进（概念依赖图中下一节点）。  
2) **强制调研**：执行 2.2。  
3) **信息量加厚**：本章必须比上一章“更厚”（至少 3 倍），衡量方式推荐：
   - 规则/卡片/例子/小测数量显著增加
   - 至少 1 个 Stepper 或 1 个 Selector（让读者动起来）
4) **输出**：只产出这一章（+ 必要的 manifest/index 增量更新，并回写上一章的下一章链接）。

---

## 4. 章节落地检查（发布前必过）

发布前用 `checklists/quality.md` 过一遍；若不达标，宁可减小范围，也不要硬凑“看起来很长”的正文。
