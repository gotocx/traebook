# TraeBook 章节质量清单（发布前必过）

## A. 整理（Structure）
- [ ] 本章标题清晰，能一眼看出学什么
- [ ] 本章学习目标 ≤ 3 条，且可验收（不是口号）
- [ ] 本章概念节点 ≤ 5 个，并标注依赖
- [ ] 本章有“下一章桥接”（为什么学下一章）

## B. 深度（Depth）
- [ ] 每个核心概念至少 2 层解释（直觉/规则）
- [ ] 至少 1 个常见误区（误解 → 更正 → 为什么会误解）
- [ ] 至少 1 个反例或边界情况

## C. 可学懂（Learnability）
- [ ] 章首有“本章地图”（你将获得什么能力）
- [ ] 章末有“可复述测试”（3 问）
- [ ] 语言无翻译腔：用中文自然表达

## D. 练习闭环（Practice Loop）
- [ ] 至少 2 题练习：基础题 + 迁移题
- [ ] 每题都给提示（Hint），且提示不直接给答案
- [ ] 有答案/解析（可以折叠）
- [ ] 题卡标题就是问题本身，展开区域才显示提示、答案和解析

## D.1 交互性（Meaningful Interactivity）
- [ ] 至少 2 种交互模式（Tabs/Collapse/Stepper/Checklist/Quiz）
- [ ] 每个交互都有“学习意义说明”（它帮助回答哪个学习问题）
- [ ] 无纯装饰动效（没有解释价值的动画/按钮）
- [ ] 主题切换按钮状态和页面实际主题一致，进入页面、切换后、刷新后都不反转

## E. 书籍形态（Book Form）
- [ ] 章节页版式统一：标题区、正文区、卡片区、总结区
- [ ] 可拔插：章节可独立作为二级页面打开
- [ ] 主题统一：保留原 UI 配色，日间是绿色系 light 方案，夜间是 dark 方案，对比度合格
- [ ] 无插图时已动态降级为概念图/步骤推演/误区对照等结构化模块，没有坏图、空图位或伪造 URL

## F. 分享（Share）
- [ ] 本章支持分享链接或导出状态（至少一种）
- [ ] 分享能复现：别人打开看到同一内容/参数/题目
